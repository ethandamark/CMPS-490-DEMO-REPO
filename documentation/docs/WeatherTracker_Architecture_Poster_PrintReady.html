Primary Execution Spine

Step 1

Consumer

  • Opens app and chooses live or preset location.
  • Confirms location and notification permissions.
  • Consumes weather, radar, and alert outputs.
Step 2

Android App Core

  • MainActivity drives UI state and navigation.
  • AuthenticationService restores/registers identity.
  • Cache and location updates keep UX responsive.
Step 3

FastAPI Orchestration

  • Handles register, weather proxy, location, and sync endpoints.
  • Normalizes errors and keeps response contracts stable.
  • Coordinates writes to Supabase PostgREST.
Step 4

On-Device Prediction

  • Room stores weather snapshots and history.
  • FeatureAssemblyService builds ML feature set.
  • ONNX inference emits probability and alert state.
Step 5

Cloud Persistence

  • Workers sync snapshots and model instances.
  • Supabase persists users, device state, locations, and outputs.
  • Heartbeat refreshes last_active_at and account status.
Branch A: API Sourcing
  • NWS supplies points, forecast, and active alerts.
  • RainViewer supplies radar frame metadata and tiles.
  • Open-Meteo supplies observations and forecast context.
Branch B: Fallback Path
  • If backend is down, app switches to Open-Meteo direct mode.
  • Health check runs every 10 seconds.
  • Flow returns to primary spine when backend recovers.
Branch C: Deferred Sync Path
  • WorkManager runs hourly and on Wi-Fi reconnect.
  • Snapshot and model_instance payloads are batched.
  • Retries preserve data consistency under intermittent network.

Execution Path Summary (Pseudo-Linear)

Main Spine
  • User opens app and permissions are evaluated.
  • Backend orchestration serves weather and state updates.
  • Forecast, alerts, and radar are rendered to the user.
ML Branch
  • LocationTrackingService captures hourly observations.
  • Features are assembled from Room history.
  • ONNX result drives local severe-weather notification.
Cloud Branch
  • Device/location updates persist through backend.
  • SnapshotSync + ModelInstanceSync push local data.
  • Supabase keeps cloud state aligned with device state.
Recovery Branch
  • If backend is unavailable, UI uses Open-Meteo directly.
  • Backend health is checked every 10 seconds.
  • Flow auto-returns to primary path when healthy.

API Sourcing and Cohesive Dependency View

NWS (weather.gov)

Supplies point metadata, daily/hourly forecasts, and active weather alerts via backend endpoints.

Open-Meteo

Feeds observation capture, 24-hour seeding, forecast prefetch, and full UI fallback mode.

RainViewer

Provides radar frame timing and tile templates used by map overlay playback.

Supabase PostgREST

Stores anonymous users, devices, locations, snapshots, and model_instance prediction records.
